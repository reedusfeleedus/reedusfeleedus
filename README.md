## Hi recruiters,

I ran into a small dilemma: either keep an ever-growing graveyard of old repositories on my profile (most of which no longer reflect how I think or build), or aggressively clean things up and lose visible evidence of past work.

So I decided to do something slightly ridiculous.

Inspired by Bitcoin’s use of hashing, I’ve made my very own brand new "proof of work" system.

Instead of leaving every project public forever, I keep open only a few repos that I think best represent my latest or favourite projects. For earlier work, I instead publish cryptographic hashes of the codebases.

The idea is simple: I can talk about the project, architecture, tradeoffs, mistakes, etc. If you're actually interested, I can send you the real code later, and you can verify that it matches the previously published hash. I.e. that I didn't suddenly invent the project the night before the interview.

Bizarre? Kind of. But I thought it was fun.
